## Goal

Migrate from the current shape (root = frontend, `server/` = backend, `ios/` + `android/` at root) to:

```
twirly/
‚îú‚îÄ‚îÄ apps/
‚îÇ   ‚îú‚îÄ‚îÄ web/       ‚Üê React + Vite (currently src/ + root config)
‚îÇ   ‚îî‚îÄ‚îÄ api/       ‚Üê Express API (currently server/)
‚îú‚îÄ‚îÄ packages/
‚îÇ   ‚îî‚îÄ‚îÄ shared/    ‚Üê shared Zod schemas + constants (NEW)
‚îú‚îÄ‚îÄ native/
‚îÇ   ‚îú‚îÄ‚îÄ ios/       ‚Üê Capacitor shell (currently ios/)
‚îÇ   ‚îî‚îÄ‚îÄ android/   ‚Üê Capacitor shell (currently android/)
‚îú‚îÄ‚îÄ capacitor.config.json    ‚Üê webDir ‚Üí apps/web/dist
‚îú‚îÄ‚îÄ pnpm-workspace.yaml
‚îú‚îÄ‚îÄ Makefile
‚îî‚îÄ‚îÄ package.json   ‚Üê orchestration scripts only
```

The Express API is a **peer** of the web app, not a subordinate. All three clients (web, iOS, Android) hit the same `/api/*` endpoints.

## Critical gotchas (reference)

| # | Issue | Mitigation |
|---|---|---|
| G1 | `ios/App/Podfile` hardcodes `../../node_modules/...`; `cap sync` does NOT regenerate it | Hand-edit Podfile in M4 to `../../../node_modules/...` and re-`pod install` |
| G2 | `android/capacitor.settings.gradle` is regenerated by `cap sync` | After moving `android/`, run `cap sync android` to auto-fix paths |
| G3 | `capacitor.config.json` `webDir` is relative to the config file | Update to `apps/web/dist` in M3 |
| G4 | Capacitor 7 supports `ios.path` / `android.path` in config | Use these in M4 to point at `native/ios` and `native/android` |
| G5 | Vite aliases all point to `./src` (relative to vite.config.js) | No rewrites needed ‚Äî aliases move with the config in M3 |
| G6 | `index.html` references `/src/index.jsx` (root-absolute, served via Vite project root) | Stays valid once Vite root becomes `apps/web/` |
| G7 | Tailwind `content: ['./src/**/*']` (relative to tailwind.config.js) | Stays valid post-move |
| G8 | `drizzle.config.js` + `migrate.js` use relative / `import.meta.url` paths | Safe across moves |
| G9 | `UPLOAD_DIR` defaults to `./uploads` (cwd-relative) | pnpm runs scripts in package dir ‚Üí cwd is `apps/api/` ‚Üí uploads at `apps/api/uploads/` |
| G10 | `npm run seed` uses `--env-file=server/.env` | Update to `apps/api/.env` in M2 |
| G11 | Two `package-lock.json` files, zero pnpm lock | Delete both in M1, let pnpm rebuild |
| G12 | `vercel.json` at root assumes web builds at root | Set Vercel rootDirectory to `apps/web` (or update buildCommand) in M3 |
| G13 | No CI workflows, no Husky | Clean slate |
| G14 | No cross-workspace imports today | `packages/shared/` in M5 is greenfield |
